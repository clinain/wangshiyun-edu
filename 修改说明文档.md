### 2.1 用户认证系统（`authController.js` + `Login.tsx` + `Profile.tsx`）

- **注册方式**：手机号 → 邮箱 + 邮箱验证码

- 新增 `sendVerificationCode` 接口（发送注册验证码）
- 新增 `sendLoginCode` 接口（发送登录验证码）
- 新增 `uploadAvatar` 接口（头像上传，存储到 `uploads/avatars/`）

- 登录表单支持"密码登录"和"验证码登录"两种模式切换

**前端 [`Profile.tsx`](frontend/src/pages/Profile.tsx)**：

### 2.11 CORS 配置修复

**修改文件**：[`server.js`](backend/server.js)

```javascript
// 修改前（不兼容）
app.use(cors({
    origin: process.env.CORS_ORIGIN || '*',
    credentials: true
}));

// 修改后（修复）
const defaultOrigins = NODE_ENV === 'production'
    ? (process.env.CORS_ORIGIN || 'https://wangshiyun.edu')
    : (process.env.CORS_ORIGIN || 'http://localhost:3000');
app.use(cors({
    origin: defaultOrigins,
    credentials: true
}));
```

> **注意**：浏览器规范要求 `credentials: true` 时 `origin` 不能为 `'*'`，必须指定具体域名。

## 六、部署注意事项

1. **邮件服务**：需要配置阿里云 DirectMail 的 AccessKey，否则验证码功能不可用
2. **数据库**：服务器启动时会自动初始化表结构（`init_sqlite.sql`），无需手动执行 SQL
3. **CORS**：生产环境需配置 `CORS_ORIGIN` 为实际域名
4. **新依赖**：前后端都需要重新 `npm install`
5. **Puppeteer**：新增的 `puppeteer` 包较大（约 300MB），Sealos 部署时需注意磁盘空间
6. **上传目录**：确保 `uploads/avatars/` 目录存在且有写入权限
7. **前端代理端口**：开发环境代理端口从 3003 改为 3004（仅影响本地开发）

## 九、部署注意事项

1. **数据库自动初始化**：服务启动时会自动执行 [`init_sqlite.sql`](backend/database/init_sqlite.sql) 创建/更新表结构，无需手动执行 SQL
2. **邮件服务**：需要配置阿里云 DirectMail 的 AccessKey，否则注册/登录验证码功能不可用
3. **Puppeteer 依赖**：生产环境需要安装 Chromium，Sealos 容器中需确认 Puppeteer 可正常运行
4. **前端代理端口**：本地开发代理端口从 `3003` 改为 `3004`（[`frontend/vite.config.ts`](frontend/vite.config.ts:18)），生产环境不受影响
5. **管理员账号**：部署后需要手动创建管理员账号，或通过数据库直接设置 `role = 'admin'`
6. **Sealos 适配**：所有修改基于 `sealos/adaptation-20260612` 分支，已适配 SQLite + HTML/PDF 导出
